# ملخص ملفات النشر - Deployment Files Summary

هذا الملف يوضح جميع الملفات التي تم إنشاؤها لإعداد المشروع للنشر على Plesk.

## 📁 الملفات الجديدة

### 1. ملفات الإعداد الأساسية

#### `backend/passenger_wsgi.py`
- **الوظيفة**: نقطة دخول Passenger لتشغيل تطبيق FastAPI
- **الاستخدام**: يستخدمه Plesk Passenger لتشغيل التطبيق
- **ملاحظة**: لا تحتاج لتعديله إلا في حالات خاصة

#### `backend/.htaccess`
- **الوظيفة**: إعدادات Apache للـ Backend
- **الاستخدام**: يوجه طلبات API إلى Passenger
- **⚠️ مهم**: يجب تحديث `PassengerAppRoot` إلى المسار الصحيح لدومينك

#### `.htaccess` (في الجذر)
- **الوظيفة**: إعدادات Apache الرئيسية للـ Frontend
- **الاستخدام**: يوجه الطلبات إلى React Router ويدير الملفات الثابتة
- **ملاحظة**: يعمل مع React SPA بشكل صحيح

### 2. ملفات التوثيق

#### `PLESK_DEPLOYMENT.md`
- **الوظيفة**: دليل شامل ومفصل خطوة بخطوة
- **المحتوى**: جميع خطوات النشر مع شرح مفصل
- **متى تستخدمه**: عند النشر لأول مرة أو عند الحاجة لتفاصيل كاملة

#### `QUICK_DEPLOY.md`
- **الوظيفة**: دليل سريع ومختصر
- **المحتوى**: الخطوات الأساسية فقط
- **متى تستخدمه**: عند النشر السريع أو عند معرفة الخطوات

#### `DEPLOYMENT_CHECKLIST.md`
- **الوظيفة**: قائمة تحقق للتأكد من اكتمال النشر
- **المحتوى**: قائمة بجميع الخطوات مع مربعات اختيار
- **متى تستخدمه**: أثناء النشر للتأكد من عدم نسيان أي خطوة

#### `ENV_TEMPLATE.md`
- **الوظيفة**: قالب متغيرات البيئة
- **المحتوى**: أمثلة لملفات `.env` مع شرح
- **متى تستخدمه**: عند إعداد ملفات البيئة

#### `README_DEPLOYMENT.md`
- **الوظيفة**: نظرة عامة على ملفات النشر
- **المحتوى**: شرح سريع لجميع الملفات
- **متى تستخدمه**: للتعرف على الملفات المتاحة

### 3. سكريبتات البناء

#### `build_frontend.sh`
- **الوظيفة**: بناء Frontend على Linux/macOS
- **الاستخدام**: `chmod +x build_frontend.sh && ./build_frontend.sh`
- **النتيجة**: مجلد `frontend/dist` جاهز للنشر

#### `build_frontend.bat`
- **الوظيفة**: بناء Frontend على Windows
- **الاستخدام**: `build_frontend.bat`
- **النتيجة**: مجلد `frontend/dist` جاهز للنشر

### 4. التعديلات على الملفات الموجودة

#### `backend/main.py`
- **التعديل**: إضافة دعم لقراءة `ALLOWED_ORIGINS` من متغيرات البيئة
- **السبب**: لضبط CORS بشكل صحيح في الإنتاج

#### `frontend/vite.config.ts`
- **التعديل**: إضافة إعدادات البناء للإنتاج
- **السبب**: لتحسين أداء البناء

## 📋 ترتيب الاستخدام الموصى به

### للمرة الأولى:
1. اقرأ `README_DEPLOYMENT.md` للتعرف على الملفات
2. اقرأ `PLESK_DEPLOYMENT.md` بالكامل
3. راجع `ENV_TEMPLATE.md` وأعد ملفات `.env`
4. استخدم `DEPLOYMENT_CHECKLIST.md` أثناء النشر

### للتحديثات:
1. استخدم `QUICK_DEPLOY.md` للخطوات السريعة
2. راجع `DEPLOYMENT_CHECKLIST.md` للتأكد

## ⚠️ تحذيرات مهمة

1. **ملفات .htaccess**: يجب تحديث المسارات فيها حسب موقعك على Plesk
2. **ملفات .env**: لا ترفعها إلى Git (موجودة في .gitignore)
3. **SECRET_KEY**: غيّره دائماً في الإنتاج
4. **ALLOWED_ORIGINS**: حدد الدومينات المسموح بها فقط

## 🔄 التحديثات المستقبلية

عند تحديث التطبيق:
1. أعد بناء Frontend
2. ثبت المتطلبات الجديدة
3. ارفع الملفات المحدثة
4. أعد تشغيل التطبيق في Plesk

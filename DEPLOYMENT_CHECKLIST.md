# قائمة التحقق من النشر - Deployment Checklist

استخدم هذه القائمة للتأكد من اكتمال جميع خطوات النشر على Plesk.

## ✅ قبل النشر

- [ ] تم بناء Frontend بنجاح (`npm run build`)
- [ ] تم اختبار التطبيق محلياً
- [ ] تم تحديث جميع المتغيرات في ملفات `.env`
- [ ] تم تغيير `SECRET_KEY` إلى قيمة آمنة
- [ ] تم تحديث `ALLOWED_ORIGINS` في `backend/.env`
- [ ] تم تحديث `VITE_API_URL` في `frontend/.env`

## ✅ إعداد Plesk

- [ ] تم رفع جميع الملفات إلى الخادم
- [ ] تم إنشاء مجلدات `backend` و `frontend/dist`
- [ ] تم رفع ملفات `.htaccess` في الأماكن الصحيحة
- [ ] تم تفعيل Passenger في Plesk
- [ ] تم إعداد Python Application في Plesk
- [ ] تم تحديد `passenger_wsgi.py` كملف البدء

## ✅ قاعدة البيانات

- [ ] تم إعداد قاعدة البيانات (SQLite أو PostgreSQL)
- [ ] تم تحديث `DATABASE_URL` في `backend/.env`
- [ ] تم تشغيل `setup.py` لتهيئة الجداول
- [ ] تم تشغيل `seed_all_users.py` لإنشاء المستخدمين
- [ ] تم اختبار الاتصال بقاعدة البيانات

## ✅ الصلاحيات

- [ ] تم تعيين صلاحيات الملفات (644)
- [ ] تم تعيين صلاحيات المجلدات (755)
- [ ] تم تعيين صلاحيات الكتابة لمجلد `uploads`
- [ ] تم تعيين صلاحيات الكتابة لمجلد قاعدة البيانات

## ✅ المتطلبات

- [ ] تم تثبيت جميع حزم Python من `requirements.txt`
- [ ] تم التحقق من إصدار Python (3.9+)
- [ ] تم التحقق من إصدار Node.js (للبناء فقط)

## ✅ SSL/HTTPS

- [ ] تم تفعيل SSL/TLS في Plesk
- [ ] تم تثبيت شهادة SSL
- [ ] تم التحقق من عمل HTTPS بشكل صحيح

## ✅ الاختبار

- [ ] تم فتح الموقع في المتصفح
- [ ] تم التحقق من عمل Frontend
- [ ] تم اختبار API: `/api/health`
- [ ] تم اختبار API Docs: `/api/docs`
- [ ] تم اختبار تسجيل الدخول
- [ ] تم التحقق من عدم وجود أخطاء CORS

## ✅ الأمان

- [ ] تم تغيير كلمات المرور الافتراضية
- [ ] تم تعطيل وضع التطوير (Debug Mode)
- [ ] تم مراجعة إعدادات CORS
- [ ] تم تفعيل Security Headers في `.htaccess`

## ✅ النسخ الاحتياطي

- [ ] تم عمل نسخة احتياطية من قاعدة البيانات
- [ ] تم حفظ نسخة من ملف `.env`
- [ ] تم توثيق إعدادات الخادم
